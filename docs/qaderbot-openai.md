# QaderBot — OpenAI Integration Guide

## 1. What QaderBot Does

QaderBot is the official QaderTech website assistant. It answers visitor questions using **only content from the QaderTech landing page** — services, FAQ, contact info, company overview, and process steps.

It supports **Arabic and English**, responds in the user's language, and falls back gracefully when OpenAI is unavailable.

---

## 2. Architecture

```
QaderBot UI (client)
  → POST /api/qaderbot
  → retrieve.ts (keyword + synonym matching)
  → knowledge.ts (structured website content)
  → openai.ts (backend-only, if key configured)
  → respondLocal.ts (fallback if OpenAI fails or is disabled)
  → { answer, source }
```

**Sources returned:**
- `openai` — answer generated by OpenAI from retrieved context
- `local` — answer built from website knowledge directly
- `fallback` — polite message when no relevant data found

---

## 3. How to Add OpenAI Key

Add to `.env.local` (never commit this file):

```env
OPENAI_API_KEY=your_key_here
QADERBOT_AI_PROVIDER=openai
QADERBOT_MODEL=gpt-4.1-mini
```

Restart the dev server after adding variables.

**Important:** Never use `NEXT_PUBLIC_OPENAI_API_KEY`. The key must stay server-side only.

---

## 4. How to Change Model

Set `QADERBOT_MODEL` in `.env.local`:

```env
QADERBOT_MODEL=gpt-4.1-mini
```

Default is `gpt-4.1-mini` if not set.

---

## 5. How to Edit the Knowledge Base

Primary file: `src/lib/qaderBot/knowledge.ts`

Content is built from:
- `src/lib/dict.ts` — hero, about, FAQ, process, contact labels
- `src/lib/servicesData.ts` — service titles and descriptions
- `src/lib/contactInfo.ts` — emails, phone, social links, address

To add new knowledge:
1. Update the relevant source file (preferred — keeps site and bot in sync)
2. Or add a new chunk in `knowledge.ts` with `id`, `type`, `locale`, `title`, `content`

Chunk types: `company`, `service`, `faq`, `contact`, `process`, `cta`, `footer`

## 6. How Local Fallback Works

Local fallback activates when:
- `OPENAI_API_KEY` is missing
- `QADERBOT_AI_PROVIDER` is not `"openai"`
- OpenAI request fails or times out (10 seconds)

Behavior:
- Greetings → friendly QaderBot greeting
- High-confidence retrieval (≥ 0.6) → direct answer from knowledge
- Medium confidence → best-effort answer from top chunks
- Low confidence → fallback message:

  **Arabic:** ما عندي معلومات كافية عن هذا الموضوع من بيانات الموقع، لكن أقدر أساعدك بالتواصل مع فريق قادر.

  **English:** I do not have enough information about that from the website data, but I can help you contact the QaderTech team.

## 7. How to Add Thmanyah Font Files

Place your licensed font files at:

```
public/fonts/thmanyah/Thmanyah-Regular.woff2
public/fonts/thmanyah/Thmanyah-Bold.woff2
```

CSS is configured in `src/app/globals.css`. Until files are added, the site uses fallback fonts:
- Arabic: IBM Plex Sans Arabic, Noto Sans Arabic, system-ui
- English: Poppins (via next/font/google), Montserrat, Inter

**Do not commit font files unless your license allows redistribution.**

## 8. Deployment Notes

### Vercel / Netlify
Add environment variables in the dashboard:
- `OPENAI_API_KEY`
- `QADERBOT_AI_PROVIDER=openai`
- `QADERBOT_MODEL=gpt-4.1-mini`
- `GOOGLE_SHEET_WEBHOOK_URL` (contact form)

### Rate Limiting
Current MVP uses in-memory rate limiting (20 requests/minute/IP). This resets on server restart and does not work across multiple instances. For production at scale, replace with Redis (see TODO in `rateLimit.ts`).

### Security
- API key never exposed to browser
- Message length limited to 500 characters
- No user message logging in production
- Chat messages rendered as plain text (no HTML)

## 10. Security Notes

- OpenAI is called **only** from `src/app/api/qaderbot/route.ts` via `src/lib/qaderBot/openai.ts`
- Never add `NEXT_PUBLIC_` prefix to OpenAI variables
- Never log full user messages in production
- OpenAI system prompt forbids inventing contact details, prices, or services
- Input sanitized and length-limited server-side
